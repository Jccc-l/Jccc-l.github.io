Hexo博客的搭建和部署

Hexo安装

准备工作

安装nodejs

首先需要安装nodejs.js和npm（nodejs的包管理器）以及版本控制工具git（为了将博客部署到GitHub上）

Linux
Arch系
  • 安装
    • sudo pacman -S nodejs npm git
Debian系
  1. 更新本地apt索引
    • sudo apt-get update
  2. 安装
    • sudo apt-get install nodejs npm git
Fedora系
  • 安装
    • sudo dnf install nodejs npm git
CentOS
  • 安装
    • sudo yun install nodejs npm git
Windows端

Windows端可以通过官网进行下载和安装git、Node.js和npm

查看git、nodejs和npm版本

  • node -v
  • npm -v
  • git -v

配置cnpm

由于npm的服务器在国外，在国内访问的速度比较慢，通过cnpm使用国内镜像源，提升下载包的速度

使用cnpm命令行工具替代默认的npm：

安装hexo

通过cnpm全局安装hexo

  • sudo cnpm i -g hexo-cli

hexo博客的建立和管理

新建并进入一个用于存储博客的文件夹此处用blog文件夹为例

mkdir ~/blog && cd ~/blog

初始化博客

创建好博客以后，会有一片hexo官方的博客hello-world

使用记事本或者编辑器编辑_config.yml，将post_asset_folder:false中的false改为true，该设置使hexo在创建一篇博客同时创建一个文件夹以存储这个博客的图片等

新建一篇博客：

进入source/_posts目录中即可查看当前存在的博客文件（以md文件形式存在）。

通过md文件对博客进行编辑

编辑完成后，在blog目录中输入hexo clean && hexo genearte(或hexo cl && hexo g)生成博客。

输入hexo s打开博客预览端口4000，在浏览器中输入localhost:4000即可预览博客

将博客部署到Github上

创建一个Github账号

通过Github官网创建一个Github账号
点击右上角Sign up根据提示进行操作即可（我相信你们≖‿≖✧）

配置git

配置好本地的信息

在Git中缓存GitHub凭据（可以用GitHub CLI或者Git Credential Manager进行配置，具体步骤可以查看GitHub的官方文件

缓存GitHub凭据以后，可以免去推送我们的博客时输入账号密码的步骤

创建一个Github仓库用于部署我们的博客

将仓库名命名为username.github.io，注意username与前面Owner中的内容保持一致，以后将会使用仓库名来访问你的博客

正常来说会显示绿色的username.github.io is available，但是我已经创建过了，就显示The repository Jccc-l.github.io already exists on this account.这样点击下面的Create repository即可创建仓库

进入你的仓库，点code，点击连接链接右边的按钮来复制仓库 url

通过cnpm安装hexo-deployer-git
